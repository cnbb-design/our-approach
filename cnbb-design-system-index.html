Foundation

Design Philosophy

Design as Engineering

This design system treats aesthetic decisions not as matters of taste but as engineering problems — constrained, measurable, and composable. It is built on the observation that the three most influential design systems of the current era — Google's Material Design, IBM's Carbon, and Shopify's Polaris — were developed independently and yet converged on the same fundamental architecture. That convergence is evidence of underlying mathematical structure, not fashion.

Every decision in this system derives from a small set of principles: perception is logarithmic, so scales must be geometric. Colour harmony requires perceptual uniformity, so the colour space must be OKLCH. Spatial relationships carry structural meaning, so spacing must encode hierarchy. And a design system is not a template applied three times — it is the set of shared invariants that allows different expressions to be recognisably part of the same family.

The seven principles that govern this system are documented in the manifesto. They are, in brief:

I Multiply, Never Add
II Perception Is the Only Coordinate System
III Name the Role, Not the Value
IV Constraint Liberates
V Unity Is Not Uniformity
VI Proximity Carries Meaning
VII Design the Invariants, Not the Instances
